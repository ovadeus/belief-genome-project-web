
================================================================================
  WHOO RU — FULL REPLIT BUILD PROMPT
  Comprehensive Site Specification v1.0
================================================================================

Build a full-stack website called "WhooRU" — a platform presenting a
self-knowledge framework, desktop application, and research publication.
The site must be production-quality, mobile-responsive, and visually
striking with a dark, premium aesthetic (deep navy/black backgrounds,
electric blue accents, clean sans-serif typography).

The name "WhooRU" is a reference to the dramatic moment in The Wizard of Oz
when the Wizard turns to Dorothy and her companions and demands to know:
"Who are you?" — a stark, startling reminder that facing and expressing
who we truly are is one of the most confronting experiences a human being
can have. The brand is serious yet playful. Pronounced "Hoo-Roo."

--------------------------------------------------------------------------------
TECH STACK
--------------------------------------------------------------------------------

- Frontend:   React (with React Router for navigation)
- Backend:    Node.js + Express
- Database:   PostgreSQL (via Replit's built-in DB or Neon)
- Auth:       JWT-based session auth for admin area
- Email:      Nodemailer or Sendgrid for subscription confirmations
- Animation:  Three.js r158+ with EffectComposer + UnrealBloomPass
- Charting:   D3.js (for belief network globe visualization)
- Uploads:    Multer (for blog featured images)
- Styling:    Tailwind CSS
- Deployment: Replit always-on

HERO ANIMATION — THREE.JS TRIPLE HELIX
--------------------------------------------------------------------------------

The homepage hero section must feature a full-screen, real-time 3D triple
helix animation built with Three.js. This is the visual centerpiece of the
entire site and must feel cinematic, scientific, and alive.

STRUCTURE:
  - Three intertwining DNA-style helical strands (three — not two —
    referencing the app's Triple Helix visualization system)
  - Each strand: ~120 luminous node spheres connected by thin cylindrical
    or line segments
  - Cross-bridge connectors between strands at regular intervals, like base
    pairs — these pulse with traveling light periodically
  - The entire helix rotates slowly on its vertical Y-axis (~0.3 RPM),
    giving a majestic, unhurried sense of scale
  - Subtle upward drift — the helix slowly rises through space, looping
    seamlessly

COLORS:
  Strand 1:        #6c8fff  (electric blue)   — Cognitive dimensions
  Strand 2:        #a78bfa  (soft violet)     — Emotional dimensions
  Strand 3:        #22d3ee  (cyan / teal)     — Philosophical dimensions
  Cross-bridges:   #ffffff at 30% opacity, pulsing to 80% in a traveling
                   wave that moves upward along the helix
  Background:      #0a0a0f with subtle radial glow at center (blue, ~8% opacity)

PARTICLE FIELD:
  - 300–500 tiny floating particles scattered in the background
  - Size: 1–2px, white at 15–25% opacity
  - Slow random drift — creates a "deep space / quantum field" atmosphere

LIGHTING & GLOW:
  - Each node sphere: bloom/glow effect via Three.js UnrealBloomPass
    (EffectComposer post-processing pipeline)
  - Bloom settings: strength 1.2, radius 0.4, threshold 0.3
  - Glow intensifies on the traveling cross-bridge pulse wave
  - No hard shadows — everything emissive/additive

CAMERA:
  - Slight auto-orbit: camera circles the helix very slowly (2–3°/second)
  - Tilt: camera sits slightly below center looking up — monumental quality
  - Mouse parallax: helix tilts slightly toward cursor position (max 5°),
    making it feel responsive and alive
  - Field of view: 60 degrees

SCALE & POSITIONING:
  - Desktop: helix occupies right 60% of hero
  - Left side: dark gradient fade (#0a0a0f → transparent) for text legibility
  - Mobile: helix centered, reduced scale, text overlaid with gradient
  - Helix extends beyond viewport top and bottom — implying infinite scale

SUBTLE DETAIL — NEURON FIRE:
  - Every 8–12 seconds, one random node on any strand briefly "activates":
    flares to full white brightness, sends a ripple of increased glow along
    its strand for ~1.5 seconds, then fades back
  - Timing is organic and unpredictable — not mechanical or looping

BACKGROUND DETAIL:
  - Very faint wireframe sphere (opacity ~5%) behind the helix, rotating
    in the opposite direction — gives depth, implies the helix exists
    inside a larger system

HELIX GEOMETRY — generate programmatically:
  x = radius * cos(t + phaseOffset)
  y = t * heightScale
  z = radius * sin(t + phaseOffset)
  phaseOffset differs by (2π/3) for each of the three strands
  Cross-bridges drawn between corresponding t-values on adjacent strands
  every N steps

PULSE WAVE:
  Animate traveling pulse by cycling emissive intensity of cross-bridge
  meshes using a sine wave offset by their index position

PERFORMANCE:
  - Use requestAnimationFrame with delta time for consistent speed
  - Reduce particle count to 150 on mobile
  - Detect low-performance devices (navigator.hardwareConcurrency < 4)
    and fall back to a CSS-animated static helix illustration
  - Target 60fps on modern hardware, graceful degradation to 30fps

LOADING STATE:
  - Before Three.js initializes: CSS pulsing placeholder — three vertical
    lines in brand colors (#6c8fff, #a78bfa, #22d3ee) with breathing
    opacity animation
  - Fade canvas in over 600ms once scene is ready

SCROLL BEHAVIOR:
  As the user scrolls down from the hero, the helix gradually shrinks and
  migrates to a small persistent position in the top-right corner of the
  viewport (like a living logo), scaled down to ~80px, continuing to
  rotate. This uses a scroll event listener + CSS transform transition.
  On mobile this persistent helix is hidden to preserve performance.

GLOBAL FEATURES & BEHAVIOR
--------------------------------------------------------------------------------

Forms:
  - Client-side + server-side validation on all forms
  - Duplicate email prevention with user-friendly message
  - Success/error toast notifications (top-right, auto-dismiss 4s)
  - Rate limiting on all form submissions (express-rate-limit: 5 req/15min)

Blog:
  - Hashtags stored as array, rendered as clickable pills
  - Clicking a hashtag filters the blog index
  - Slugs: auto-generated from title (kebab-case), editable in admin
  - Read time: auto-calculated from word count (avg 200 wpm)

SEO:
  - Each page: unique <title> and <meta description>
  - Blog posts: Open Graph tags (og:title, og:description, og:image)
    for rich social sharing previews
  - Canonical URLs on all pages
  - robots.txt: allow all except /admin

Mobile:
  - Fully responsive at 320px, 768px, 1024px, 1440px breakpoints
  - Hamburger navigation on mobile
  - Touch-friendly tap targets (min 44px)

Security:
  - All /admin routes protected by JWT middleware
  - Rate limiting on login endpoint (5 attempts / 15min, then lockout)
  - Input sanitization on all text fields (prevent XSS)
  - SQL injection prevention via parameterized queries
  - CORS configured for production domain only
  - Admin password stored as bcrypt hash (rounds: 12)
  - httpOnly, Secure, SameSite=Strict cookies for JWT
